context: "CareerCraft is an all-in-one career-development platform for tech students and professionals, bringing together portfolio hosting, resume building, a GitHub README editor, an AI-powered mock interview system, weekly coding contests, learning resources, and placement support into a single ecosystem. Built by a student team and later adopted by KLH University (Hyderabad) to support its students' placement journeys. I joined the team for about a month to restructure the platform's information architecture and lead a complete usability/UX-focused redesign across the whole product, along with some UI theme work — primarily in Figma."
scope: Information architecture restructuring and end-to-end UX redesign across the full platform, plus UI theme work.
duration: 1 Month
role: UX/UI Designer

outcomes:
  - shape: triangle-up
    label: 9 platform modules redesigned
  - shape: circle
    label: Adopted by KLH University
  - shape: triangle-down
    label: 1 Month IA + UX overhaul
process:
  - problem: Nine feature areas, no coherent structure
    description: CareerCraft had grown to span portfolios, resumes, a GitHub README editor, learning resources, AI mock interviews, weekly contests, placement support, and certificate verification — each useful on its own, but without an information architecture tying them together. Restructured the IA platform-wide so the path through the platform's core loop (learn, build, showcase, get placed) was legible instead of something users had to piece together themselves.
  - problem: Usability hadn't kept pace with feature growth
    description: Led a complete usability-focused redesign across the product, auditing existing flows and rebuilding them around clearer, more consistent interaction patterns — the kind of pass a platform needs once it's accumulated features faster than it's accumulated design decisions.
  - problem: The UI needed a consistent visual language
    description: Alongside the structural work, did UI theme work to bring a more consistent visual language across the platform's modules, working primarily in Figma.
learnings:
  - With nine distinct feature areas, restructuring the IA meant designing the connections between modules, not just each module on its own.
  - A month is enough time to fix structure and usability, but you don't see whether it actually worked until real users adopt it — like KLH University did.
  - Working on a live, adopted platform is different from a case-study project — the redesign has to hold up for real students, not just read well.
